ENUNCIADO¶

Somos trabajadores de una financiera de consumo de uno de los cinco bancos ms grandes de Espaa. Desde el departamento de operaciones nos informan de que estn teniendo problemas con las tarjetas de crdito. Muchas tarjetas estn siendo duplicadas y hay una gran cantidad de transferencias fraudulentas que se estn cometiendo. Para resolver este hecho preguntan si podemos ayudarles con algn modelo estadstico.

Uno de los problemas en la financiacin al consumo es el fraude en tarjetas, dinero que es cargado a los clientes por productos que ellos no han adquirido. Esto es un problema importante para el sector financiero, ya que, adems de las prdidas monetarias, disminuye la fidelizacin de los clientes. Hay que recordar que el sistema monetario actual se basa en la confianza, y el dinero es lo primero que escapa cuando no hay confianza en las entidades.

CONTEXTO¶

En un escenario del mundo real, los algoritmos no supervisados se utilizan en estos casos para filtrar del total de transacciones cules son anmalas. A veces se utilizan reglas de negocio para hacer ese filtrado, pero muchas veces esas reglas son muy difciles de formular. Ya que no es viable analizar todas las transacciones, un modelo no supervisado que filtre comportamientos extraos es muy til para realizar una primera criba. Para ello se hallan caractersticas de cada transaccin (X) y con detectores de anomalas se estima cules son anmalas y cules no.

Posteriormente, se auditan todas esas transacciones anmalas y se obtienen las etiquetas reales (Y). Se halla qu transacciones son fraudulentas (Y=1) y cules no (Y=0).

Llegado un punto, la entidad financiera tiene datos etiquetados (X,Y) que puede utilizar para realizar un clasificador de aprendizaje supervisado.

PLANTEAMIENTO DEL PROBLEMA¶

En este caso se tienen las caractersticas de cada transaccin (X) y la etiqueta indicando si son fraude o no (Y). Se podra realizar directamente un clasificador de aprendizaje supervisado, pero el objetivo es tener una aproximacin a cmo de til puede ser un modelo de deteccin de anomalas en pasos previos.

Se va a suponer que no existe una etiqueta (Y), y se va a realizar un modelo de deteccin de anomalas solamente con las caractersticas (X). Se hallar qu transacciones se obtienen como anmalas y luego se compararn con las etiquetas reales (Y). De este modo se podr ver cmo un detector de anomalas podra servir en ese primer filtrado.

Detección de anomalías¶

Despus de ajustar el modelo Local Outlier Factor (LOF) a los datos, este fragmento de cdigo calcula los factores de anomala para cada punto en el conjunto de datos. Los valores ms negativos indican una mayor anormalidad, es decir, son ms propensos a ser anomalas.

In [ ]:
factores_LOF = clf.negative_outlier_factor_
factores_LOF
Out[ ]:
array([-1.19618609, -1.15222336, -1.61787169, ..., -1.32806678,
       -1.24067801, -1.75327498])

El modelo nos proporciona lo que se denominan factores de anomalas negativos. Cuanto ms alto es este valor, ms normal es el punto. Es decir, si queremos quedarnos con el 2 % de puntos ms anmalos, debemos quedarnos con el 2 % de valores ms bajo. Ponemos el umbral en un 2 % para tener margen.

In [ ]:
# Clasificamos los puntos de acuerdo con sus factores de anomala calculados por el modelo LOF. 
y_pred = factores_LOF.copy()
y_pred[factores_LOF >= np.percentile(factores_LOF,2.)] = 0
y_pred[factores_LOF < np.percentile(factores_LOF,2.)] = 1

En resumen, este cdigo clasifica los puntos como anomalas (1) o no anomalas (0) basndose en sus factores de anomala calculados por el modelo LOF, utilizando el percentil 2 como umbral para distinguir entre anomalas y no anomalas. Los puntos con factores de anomala ms bajos que el percentil 2 se consideran anomalas, mientras que los puntos con factores de anomala iguales o superiores al percentil 2 se consideran no anomalas.
